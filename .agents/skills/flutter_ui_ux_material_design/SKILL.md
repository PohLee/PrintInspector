# Flutter UI/UX & Material Design 3 Skill

This skill provides a comprehensive guide to building state-of-the-art Flutter applications that adhere to Material Design 3 (M3) principles while maintaining a premium, modern aesthetic.

## 🎨 Core Principles: Material Design 3 (Material You)

Material Design 3 is the latest version of Google's open-source design system. It focuses on personalization, adaptability, and expressive interfaces.

### 1. Dynamic Color & Theming
M3 introduces a dynamic color system that can derive colors from a user's wallpaper or a specific seed color.
- **`ColorScheme.fromSeed`**: Use this to generate a harmonious palette.
- **Surface Tints**: M3 uses surface tints instead of elevation shadows to show hierarchy.
- **Dark Mode Support**: Always implement a consistent dark theme using `ThemeData.dark()`.

```dart
ThemeData(
  useMaterial3: true,
  colorScheme: ColorScheme.fromSeed(
    seedColor: const Color(0xFF6750A4),
    brightness: Brightness.light,
  ),
)
```

### 2. Typography Hierarchy
M3 simplifies typography into a clear hierarchy. Use `Theme.of(context).textTheme` to ensure consistency.
- **Display**: Large, expressive text for headers.
- **Headline**: For section titles.
- **Title**: For small headlines or list items.
- **Body**: For primary text content.
- **Label**: For small components like buttons or captions.

### 3. Layout and Adaptability
Design for all screen sizes (Mobile, Tablet, Desktop).
- **`LayoutBuilder`**: Use for widget-specific constraints.
- **`MediaQuery`**: Use for global screen information.
- **Navigation Rail vs. Navigation Bar**: Use `NavigationRail` for wide screens and `NavigationBar` for mobile.

## ✨ Premium Aesthetics & WOW Factor

To create a "WOW" experience, go beyond basic Material widgets:

### 1. Modern Gradients & Glassmorphism
- Use subtle linear or radial gradients for backgrounds.
- Implement **Glassmorphism** using `BackdropFilter` and `ImageFilter.blur`.

```dart
ClipRRect(
  borderRadius: BorderRadius.circular(20),
  child: BackdropFilter(
    filter: ImageFilter.blur(sigmaX: 10, sigmaY: 10),
    child: Container(
      color: Colors.white.withOpacity(0.1),
      child: child,
    ),
  ),
)
```

### 2. Micro-Animations
- Use **Implicit Animations** (`AnimatedContainer`, `AnimatedOpacity`) for smooth transitions.
- Implement **Hero** animations for shared elements between screens.
- Use **Lottie** or **Rive** for complex, high-performance vector animations.

### 3. Custom Shadows & Elevation
- While M3 favors surface tints, subtle custom shadows can add depth and a "premium" feel.
- Avoid harsh black shadows; use semi-transparent versions of the primary color.

## 🛠 Best Practices

1.  **Semantic HTML/Widgets**: Use appropriate semantic widgets (e.g., `Semantics`) for accessibility.
2.  **Performance**: Avoid unnecessary rebuilds by using `const` constructors and `RepaintBoundary`.
3.  **State Management**: Use clean architectures (BLoC, Provider, Riverpod) to separate UI from logic.
4.  **Haptics**: Add subtle haptic feedback using `HapticFeedback` for a tactile feel.
